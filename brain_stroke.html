Analyse de données - Brain Stroke
¶

Idir SADAOUI, Hamady CISSÉ

Un Brain Stroke (ou accident vasculaire cérébral en français) est l’arrêt brutal de la circulation sanguine au niveau d’une partie du cerveau).
Il y a deux types d'AVC :

  • Les AVC ischémiques qui sont dus à un caillot dans une artère.
  • Les AVC hémorragiques qui sont dus à une hémorragie, par lésions des parois du vaisseau.

Le but de ce jeu de données est de prédire si un patient à eu ou non un AVC et pour cela, le jeu de données nous fournit 11 variables qui sont les
suivantes :

  • gender : genre du patient "Male" ou "Female".
  • age : l'âge du patient.
  • hypertension : variable binaire : 0 si le patient n'a pas d'hypertension, 1 si le patient a de l'hypertension.
  • heartdisease : variable binaire : 0 si le patient n'a pas de maladie cardiaque , 1 si le patient a une maladie cardiaque.
  • evermarried : "Yes" si le patient est mariée, "No" sinon.
  • worktype : Type de travail : "children", "Govtjov", "Neverworked", "Private" ou "Self-employed".
  • Residencetype : Lieu de vie "Rural" ou "Urban".
  • avgglucoselevel : taux de glucose dans le sang en mg/dl.
  • bmi : L'indice de masse corporelle en kg/m$^2$.
  • smoking status : le statut de fumeur du patient : "formerly smoked", "never smoked", "smokes" ou "Unknown".
  • stroke : variable target : 1 si le patient a eu un AVC, 0 sinon.

Pour cela nous avons choisi la normalisation MinMax grâce à la fonction MinMaxScaler de scikit-learn.
La formule utilisé par cette normalisation est la suivante :

$$ \frac{X_{i} - X_{min}}{X_{max}-X_{min}} $$

avec $X_{i}$ l'observation $i$ pour $i$ = {1,...,4981}, $X_{max}$ et $X_{min}$ respectivement les valeurs maximal et minimal de la colonne.

Entrainements et évaluations¶

Dans cette analyse, nous avons selectionner 5 algorithmes de machine learning :

  • La régression logistique
  • Les Support Vector Machine
  • Les k-plus proches voisins
  • Le perceptron multi-couches
  • Les forêts aléatoires

On va d'abord couper notre jeux de données en deux grâce à la fonction train_test_split de scikit-learn en créant un jeu de données d'entrainement et un jeu de données de test, avec un jeu de test de taille égale à 20% du jeu de données d'origine :

In [328]:
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2, random_state=25)

puis on entraine notre jeu de données d'entrainement avec tout les algorithmes :
Nous avons essayer de trouver les meilleurs hyperparamètres pour chaque modèles grâce à la fonction GridSearchCV de scikit-learn mais les hyperparamètres les plus optimaux pour chaque modèle étaient ceux par défaut.

En ce qui concerne les Perceptrons multi-couches, nous avons construit un réseau de neurones de 4 couches cachées composés de 100 neurones chacunes et un nombre d'itérations maximum de 1000 (qui n'est pas atteint car la condition que nous avons fixé à été satisfaite à savoir ici d'arrêter l'algorithme si le loss ne change pas de l'ordre de 10$^{-4}$ après 20 itérations consécutives).

Nous allons à présent calculer les différents scores de performances de chaque modèles : l'Accuracy, le Recall, la Precison ainsi que le F1-Score.
Ces métriques sont données par les formules suivantes :

$$Accuracy = \frac{Nombre \ de \ prédictions \ correctes}{Nombre \ de \ prédictions \ total} \ ; \ Recall = \frac{Vrai \ Positif}{Vrai \ Positif \ + \ Faux \ Négatif} $$


$$ Precision = \frac{Vrai \ Positif}{Vrai \ Positif \ + \ Faux \ Positif} \ ; \ F1\_Score = 2 \ * \ \frac{Recall \ * \ Precision}{Recall \ + \ Precision} $$

Avant de conclure, nous pouvons effectuer des validations croisées sur les modèles pour vérifier leur fiabilité.
Pour cela nous allons utiliser la fonction cross_val_score de scikit-learn, en coupant en 5 le jeu de données d'entrainement avec la fonction ShuffleSplit de scikit-learn qui est en fait un KFold stratifié, puis on prend à chaque fois la moyenne des scores, on a :

In [334]:
CVScores = []
for algo_nom, algo in modeles.items():
    cv = ShuffleSplit(5, test_size = 0.2)
    add = cross_val_score(algo,X_train, y_train, cv=cv).mean()
    CVScores.append(add)
    print("{} {}%".format(algo_nom,round(add*100,3)))
